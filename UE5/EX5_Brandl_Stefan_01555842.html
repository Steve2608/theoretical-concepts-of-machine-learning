Answer 5.1:

  • The VC-dimension of the hypothesis class of intervals is $2$

For $0$ and $1$ points this holds trivially.

We classify a point $x$ as $1$ $\iff$ $\exists a, b | a < x < b$ with $x \in \mathbb{R}$.

We classify a point as $0$ if it is outside this interval.

For $x_1 < x_2$ we get the following intervalls

labeling $x_1$ $x_2$ possible interval
1 0 0 $(x_1, x_2)$
2 0 1 $(x_1, \infty)$
3 1 0 $(-\infty, x_2)$
4 1 1 $(-\infty, \infty)$

For three points however, this is no longer possible

labeling $x_1$ $x_2$ $x_3$ possible interval
1 0 0 0 $(x_3, \infty)$
2 0 0 1 $(x_2, \infty)$
3 0 1 0 $(x_1, x_3)$
4 0 1 1 $(x_1, \infty)$
5 1 0 0 $(-\infty, x_2)$
6 1 0 1 impossible
7 1 1 0 $(-\infty, x_3)$
8 1 1 1 $(-\infty, \infty)$

Thus we see that the $VC_{dim}$ of intervals is $2$.

  • The VC-dimension of the hypothesis class of unions of $k$ intervals is 2$k$.

Assuming that all intervalls are well ordered and do not overlap we could (in the worst case) end up with alternating $10$ labels (in regex syntax: $(1^+0^+)^k$ one or more $1$ followed by one or more $0$, repeated $k$ times).

Then we can iteratively split one label series of either $1$s or $0$s from the union. Since there are $2$ series per set over $k$ sets in total, at worst there are $2k$ splits necessary.

Thus the $VC_{dim}$ of $k$ unions of intervals is $2k$.

For each of the answers, we need to prove the following:

  • It is possible to shatter all $2^n$-possible label encodings
  • It is not possible to shatter all $2^{n+1}$ label encodings

The VC-dimension of the hypothesis class of axis aligned rectangles is 4¶

The minimum bounding box for 4 points in 2-dimensional space is a rectancle. We can reduce the width and length of the rectancle (and even move it) to exclude any subset of points.

Thus we have shown, that $VC_{dim}(Rectangle) \ge 4$

It is, however, not possibly to exclude just the blue point from the bounding rectangle. It can be shown for every axis-aligned rectangle, that we can construct a blue point that lies within the bounding box of the rectangle.

Thus we have shown, that $VC_{dim}(Rectangle) \not> 4$ implying $VC_{dim}(Square) = 4$

The VC-dimension of the hypothesis class of axis aligned squares is $3$¶

The minimum bounding box for 3 points in two dimensional space is a square, which we can move to exclude any subset of points.

Thus we have shown, that $VC_{dim}(Square) \ge 3$

Analogously to above we cannot, however, exclude just the blue point, showing that $VC_{dim}(Square) \not> 3$, thus proving that $VC_{dim}(Square) = 3$

The VC-dimension of the hypothesis class of general triangles is greater or equal than $7$¶

If there are 7 points distributed on a circle, at worst there are 3 contiguous blocks of negative labels (we can define the smaller set of labels (minority) as negative and show that there is a triangle that includes these 3 and JUST these 3 contiguous blocks: like $\{+, -, +, -, +, -, +\}$)

The exercises should convince you about the following observations: first, simple figures, which at first sight look very similar geometrically, might lead to different properties concerning their abilities of separating data(like squares, rectangles and triangles). Secondly, if the complexity of the separating class gets higher, then this usually also leads to better separation properties as indicated in the first example. Even the number of parameters that characterize the hypothesis class is not an indicator for it's VC-dimension: in the lecture even an example of a one-parameter family with infinite VC-dimension was mentioned.

Next we want to implement a linear support-vector classifier(SVC) that separates the data "as accurately as possible", using a separating hyperplane(in our case: a straight line, as we are considering the situation in the plane). What we want to achieve in the end is that the separating line's distance from the data should be as large as possible(indicated by a large so called "margin"), so that also test data can be classified as accurately as possible. As this usually isn't possible in general situations (e.g. when a lot of randomness is present in the data), certain variables, that don't exactly fit in this framework have to be "punished" using so called slack-variables. It might e.g. be allowed for a certain amount of points to lie inside the margin or lie on the wrong side of the borders. This procedure in many situations can improve the generalization performance of the classifier. In any case, we won't go into the details here, and just mention that the amount of this "punishment" is encoded in the hyperparameter $C$ for the SVC. def k_circle_points(k):
    data = []
    for i in range(k):
        angle = np.pi * np.random.uniform(high=2)
        data.append((np.cos(angle), np.sin(angle)))
    data = np.asarray(data)
    return data, (data[:, 1] > 0).astype(int)

def compute_svm(k):
    if k <= 10:
        raise ValueError(f'k needs to be >= 10; but was: {k}')
        
    clf = svm.SVC(kernel='linear')
    margins = []
    for j in range(10, k + 1):
        X, y = k_circle_points(j)
        clf.fit(X, y)
        margins.append(1 / np.linalg.norm(clf.coef_))
    
    return margins

marg = np.asarray(compute_svm(1000))

It should be clearly visible from the plot, that, up to small fluctuations, the value of the margin decreases for a larger number of data points. This should come as no surprise, since the model gets increasingly complex for a larger number of data, and therefore a clearer separation is harder, which is indicated by smaller margins. Also the increase of the VC-dimension should come as a reasonable fact, since VC-dimension provides an indicator for model complexity and it indeed seems to be a reasonable concept that can provide a theoretical basis for measuring the quality of widle applied classifier algorithms.
